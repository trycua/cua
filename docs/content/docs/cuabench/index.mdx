# Cua Bench

A toolkit for verifiable cross-platform computer-use RL environments and benchmarks. Supports real Windows, Linux, macOS, and Android VM environments with reproducible base images.

## Quick Start

```bash
# Install
git clone https://github.com/trycua/cua-bench.git && cd cua-bench
uv tool install -e .

# Create base image
cb image create linux-docker

# Run interactive task
cb interact click-icon --dataset cua-bench-basic

# Run single task with agent
cb run task tasks/click_button --agent cua-agent --model anthropic/claude-sonnet-4-20250514

# Run entire dataset in parallel
cb run dataset datasets/cua-bench-basic --agent cua-agent --max-parallel 4
```

## Key Features

- **Base Images** - Reproducible VM images for Windows, Linux, macOS, and Android
- **Task Authoring** - Python API with async decorators for setup, evaluation, and oracle solutions
- **Custom Agents** - Docker image agents for production, import path agents for development
- **2-Container Architecture** - Agent and environment containers communicate via Docker network
- **Benchmark Execution** - Run evaluations locally or at scale on cloud
- **Training Data** - Export trajectories to datasets for UI grounding models

## Architecture

Cua-Bench uses a **2-container architecture** where agent and environment run in isolated containers:

```
┌─────────────────────────────────────────────────────────────┐
│  Docker Network: cua-task-{id}                              │
│                                                             │
│  ┌─────────────────────┐    ┌─────────────────────────────┐│
│  │  cua-agent-{id}     │    │  cua-env-{id}               ││
│  │                     │    │                             ││
│  │  Your agent image   │───►│  Base image environment     ││
│  │  (or built-in)      │HTTP│  (linux/windows/android)    ││
│  │                     │    │                             ││
│  └─────────────────────┘    └─────────────────────────────┘│
└─────────────────────────────────────────────────────────────┘
```

This architecture is consistent between local development and cloud deployments.
