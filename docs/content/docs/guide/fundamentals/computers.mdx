Cua supports multiple sandbox providers, each with different trade-offs. Choose based on your requirements for OS support, isolation, and deployment environment.

## Provider Comparison

| Provider | OS | Host Requirements | Isolation | Persistence | Best For |
|----------|-----|-------------------|-----------|-------------|----------|
| **Cloud** | Linux, Windows, macOS | Any | Full VM | Configurable | Production, teams, CI/CD |
| **Docker** | Linux | Docker installed | Container | Volume mounts | Local dev, testing |
| **Lume** | macOS | macOS host | Full VM | Snapshot-based | macOS automation |
| **Windows Sandbox** | Windows | Win 10 Pro/11 | Sandbox | Ephemeral only | Windows automation |
| **QEMU Docker** | Linux, Windows, Android | Docker + KVM | Full VM | Disk images | Multi-OS testing |
| **Host** | Any | None | None | N/A | Debugging (not recommended) |

## Provider Details

### Cloud (`provider_type="cloud"`)

Managed VMs hosted by Cua. No local setup required.

- **OS options**: Linux (Ubuntu), Windows, macOS
- **`provider_type`**: `"cloud"`
- **Required params**: `name`, `api_key` (or `CUA_API_KEY` env var)
- **Persistence**: Sandbox state persists between sessions unless explicitly reset

### Docker (`provider_type="docker"`)

Linux desktops running in Docker containers.

- **OS options**: Linux only (XFCE or Ubuntu/KASM)
- **`provider_type`**: `"docker"`
- **Required params**: `image`
- **Images**: `trycua/cua-xfce:latest` (lightweight), `trycua/cua-ubuntu:latest` (full desktop)
- **Persistence**: Ephemeral by default, use volume mounts for persistence

### Lume (`provider_type="lume"`)

Native macOS VMs using Apple's Virtualization framework.

- **OS options**: macOS only
- **`provider_type`**: `"lume"`
- **Required params**: `name` (image name)
- **Host requirement**: macOS with Lume CLI installed
- **Persistence**: Snapshot-based, can restore to clean state

### Windows Sandbox (`provider_type="winsandbox"`)

Built-in Windows isolation feature.

- **OS options**: Windows only
- **`provider_type`**: `"winsandbox"`
- **Host requirement**: Windows 10 Pro/Enterprise or Windows 11
- **Persistence**: Always ephemeral (resets on close)
- **Additional dep**: `pywinsandbox`

### QEMU Docker (`provider_type="qemu"`)

Full VMs running inside Docker via QEMU emulation.

- **OS options**: Linux, Windows, Android
- **`provider_type`**: `"qemu"`
- **Required params**: `image` (golden image path)
- **Host requirement**: Docker with KVM support
- **Persistence**: Disk image based

### Host Desktop (`use_host_computer_server=True`)

Direct control of your machine. **Use with caution.**

- **OS options**: Whatever your host runs
- **No isolation**: Agent actions affect your real system
- **Use case**: Debugging only, never in production

## Quick Reference

```python
from computer import Computer

# Cloud (recommended for production)
computer = Computer(os_type="linux", provider_type="cloud", name="my-sandbox", api_key="...")

# Docker (recommended for local dev)
computer = Computer(os_type="linux", provider_type="docker", image="trycua/cua-xfce:latest")

# Lume (macOS only)
computer = Computer(os_type="macos", provider_type="lume", name="macos-sequoia-cua:latest")

# Windows Sandbox (Windows host only)
computer = Computer(os_type="windows", provider_type="winsandbox", ephemeral=True)

# Host (debugging only)
computer = Computer(use_host_computer_server=True)
```

For setup instructions, see [Set Up a Sandbox](/docs/guide/get-started/set-up-sandbox).
