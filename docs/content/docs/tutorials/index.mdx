Tutorials teach Cua by building something end to end. Start here if you are new to Cua. Each tutorial is *one happy path* you can follow from start to finish.

- [Drive your first app](/tutorials/drive-your-first-app) - operate a native app in the background with Cua Driver.
- [Your first cloud sandbox](/tutorials/your-first-cloud-sandbox) - spin up a disposable cloud computer.
- [Run code and drive the GUI in a sandbox](/tutorials/run-an-agent-in-a-sandbox) - run code and drive the GUI in a cloud sandbox via the SDK.
- [Create a computer-use sandbox for Hermes + Cua Driver](/tutorials/sandbox-for-hermes-and-cua-driver) - drive a sandbox with the Hermes model through Cua Driver.
- [Create a computer-use sandbox to surf the web](/tutorials/sandbox-to-surf-the-web) - build a sandbox with a browser and complete a web task.
- [Computer-use sub-agent with MCP sampling](/tutorials/sub-agent-with-mcp-sampling) - have a sub-agent request model completions through MCP sampling.
- [Benchmark Codex and Claude Code with Cua Bench](/tutorials/benchmark-with-cua-bench) - score two coding agents on the same computer-use task.
- [RL a model for computer-use tasks](/tutorials/rl-a-model-for-computer-use) - train a model on computer-use tasks with sandboxes as environments.

Once you are comfortable, [How-to guides](/how-to-guides) cover specific goals and [Reference](/reference) has the exact APIs.
