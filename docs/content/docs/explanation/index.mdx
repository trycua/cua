Use these pages when you want the model behind Cua rather than a step-by-step guide or API table. They explain how Cua Driver and Cua Sandbox differ, why Cua Driver tries to keep the desktop usable while it acts, and where each component fits in the stack.

Start with [What is computer use?](/explanation/what-is-computer-use) for the basic model. Read [Best-effort background](/explanation/the-no-foreground-contract) to understand Cua Driver's default behavior on a shared machine, then [Capture and delivery modalities](/explanation/capture-and-delivery-modalities) for the action axes. [Process model](/explanation/process-model), [How sandboxes work](/explanation/how-sandboxes-work), and [Architecture](/explanation/architecture) fill in the runtime pieces.
