# Simulated Tasks

Simulated tasks use synthetic HTML webapps to test agent capabilities in controlled environments. They run via Playwright in a headless browser, making them fast and lightweight.

## What are Simulated Tasks?

- **Lightweight**: Run in `linux-docker` containers, no KVM required
- **Fast**: Quick to set up and iterate (~2 seconds startup)
- **Deterministic**: Consistent behavior for reproducible testing
- **No Docker Required**: Uses Playwright directly

## Available Datasets

| Dataset | Tasks | Description |
|---------|-------|-------------|
| `cua-bench-basic` | 13 | Core UI interactions (click, type, drag, etc.) |
| `cua-bench-forms` | - | Form filling and validation |
| `cua-bench-navigation` | - | Multi-page navigation |

## Running Simulated Tasks

```bash
# Check task provider type
cb task info ./tasks/my_task

# Interactive mode (opens Chromium window)
cb interact ./tasks/slack_env --oracle

# Run with agent
cb run --dataset cua-bench-basic --agent cua-agent
```

## When to Use

- Agent development and debugging
- CI/CD pipeline testing
- Quick iteration on agent logic
- Testing UI interaction patterns
