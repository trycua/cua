This guide walks you through creating and running your first macOS VM with Lume.

<Callout type="warn">
  **Registry Maintenance Notice**: The prebuilt images in the [trycua registry](https://github.com/orgs/trycua/packages) are no longer actively
  maintained. For new setups, we recommend building custom images using the [Unattended Setup](/lume/guide/fundamentals/unattended-setup) feature
  to configure macOS VMs according to your specific needs.
</Callout>

## Create a Custom VM

For more control, create a VM from scratch:

```bash
# Create a new macOS VM using the latest IPSW
lume create my-vm --os macos --ipsw latest

# Run the VM
lume run my-vm
```

### Customize Resources

```bash
# Create with custom resources
lume create my-vm --os macos --ipsw latest \
  --cpu 4 \
  --memory 8GB \
  --disk-size 100GB
```

Default values:

- **CPU**: 4 cores
- **Memory**: 8GB
- **Disk**: 50GB

## Basic VM Operations

```bash
# List all VMs
lume ls

# Get VM details
lume get my-vm

# Stop a running VM
lume stop my-vm

# Delete a VM
lume delete my-vm
```

## Run Without Display

For headless operation (automation, CI/CD):

```bash
# Run without opening a display window
lume run my-vm --no-display
```

## Clone a VM

Create a copy of an existing VM:

```bash
lume clone my-vm my-vm-copy
```

## Automate VM Configuration

Use unattended setup to automatically configure a fresh VM with user account, settings, and software:

```bash
# Create and configure a VM automatically
lume create my-vm --os macos --ipsw latest \
  --username lume \
  --password lume \
  --auto-run
```

This creates the VM and runs the initial macOS setup wizard automatically. See [Unattended Setup](/lume/guide/fundamentals/unattended-setup) for more options.
